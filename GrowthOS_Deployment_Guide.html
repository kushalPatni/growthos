1. What Was Built

GrowthOS is a production-grade full-stack web application. Every piece of data is stored in a real PostgreSQL database — nothing is lost on refresh. You log in with email and password, and your data is private to your account.

LayerTechnologyWhat it does
FrontendReact 18 + ViteAll the pages, UI, charts, calendar
BackendNode.js + ExpressREST API — handles all data operations
DatabasePostgreSQL 14+Stores users, tasks, logs, notes, goals, everything
AuthJWT + bcryptSecure login; passwords are hashed, never stored plain
ChartsChart.js + react-chartjs-2All analytics charts

You need these three tools installed on your computer before anything else works.

4. Local Setup — Step by Step

1
Place all the project files in a folder called growthOS on your computer, matching the structure shown in Section 2 exactly.
2
Open a terminal (Command Prompt on Windows, Terminal on Mac/Linux) and navigate into the folder:
cd path/to/growthOS
3
Install backend dependencies:
cd backend
npm install
cd ..
4
Install frontend dependencies:
cd frontend
npm install
cd ..
5
Install root dependencies (concurrently — for running both servers at once):
npm install
6
Create your .env file — copy the example and edit it:
# On Mac/Linux:
cp backend/.env.example backend/.env

# On Windows (Command Prompt):
copy backend\.env.example backend\.env
Now open backend/.env in any text editor (Notepad, VS Code, etc.) and fill in your values — see Section 5 and 11.

5. Database Setup

Step 1 — Confirm PostgreSQL is running

# Mac:
brew services list | grep postgresql

# Ubuntu:
sudo service postgresql status

# Windows: Check Services app — look for "postgresql-x64-16"

Step 2 — Create the database

# Open the PostgreSQL command line:
psql -U postgres

# You will be asked for the postgres user password (set during install)
# Once inside psql, type:
CREATE DATABASE growthos;
\q
On Ubuntu you may need to run: sudo -u postgres psql instead

Step 3 — Fill in your .env file

Open backend/.env and set it up like this:

PORT=5000
DATABASE_URL=postgresql://postgres:YOUR_POSTGRES_PASSWORD@localhost:5432/growthos
JWT_SECRET=make_this_a_long_random_string_at_least_32_chars
NODE_ENV=development
FRONTEND_URL=http://localhost:5173
Replace YOUR_POSTGRES_PASSWORD with the actual password you set when installing PostgreSQL. Never share or commit this file.

Step 4 — Run the database migration

cd backend
npm run db:migrate
cd ..

You should see: ✅ Database migration complete!

This creates all 10 tables in your database. You only need to run this once. The tables created are:

  • users — accounts with bcrypt-hashed passwords
  • tasks — tasks with parent_id self-reference for infinite nesting
  • projects — project containers linked to tasks
  • daily_logs — daily study hours, mood, and notes (one per day per user)
  • notes — personal notes with pin and search
  • goals — top-level roadmap goals
  • roadmap_items — months, weeks, day topics under each goal
  • revision_topics — spaced repetition topics with box level and next due date
  • time_logs — timer session seconds per task per day
  • calendar_events — custom calendar events with colour coding

6. Running the App

Start everything with one command

From the root growthOS/ folder:

npm run dev

This starts both servers simultaneously:

Open http://localhost:5173 in your browser. You should see the GrowthOS login screen. Register an account and you're ready to go!

Or run them separately in two terminals

cd backend
npm run dev
# Shows: 🚀 GrowthOS backend running on port 5000

# Terminal 2 — Frontend
cd frontend

# Shows: Local: http://localhost:5173

Stopping the app

Press Ctrl + C in the terminal to stop.

7. How to Use Every Feature

Dashboard

  • Shows your daily metrics: tasks done today, hours studied, streak, completion rate
  • Mini bar chart shows your study hours for the last 7 days
  • The calendar on the right shows the current month and changes automatically as months pass
  • Blue dots on calendar dates = tasks with that deadline. Coloured dots = custom events
  • Click any calendar date to instantly see what tasks are due that day
  • Log your mood by clicking one of the emoji buttons (saves to database)
  • Click ⚡ Log Day in the top bar to log today's hours, what you studied, and wins
  • Progress bars under the streak show your completion rate per subject tag

Tasks — The Core System

  • Click + New Task to add a task. Fill in: title, description, tag, priority, deadline, planned hours, and optionally link it to a project
  • Tasks are stored permanently in PostgreSQL — they survive refresh and reopening the browser
  • Click the ▸ arrow on the left of any task to expand it
  • Once expanded, type in the "+ subtask" input at the bottom and press Enter to add a subtask inside it
  • Expand that subtask and add another subtask inside it — this nesting goes as deep as you want
  • A parent task shows a done/total counter (e.g. 2/5) showing how many subtasks are complete
  • Click 📝 to open a notes area attached to any task
  • Click the done checkbox to toggle a task complete — it updates in the database immediately
  • Use the search bar and filters to find tasks by name, tag, or status
Nested tasks example:
"Learn Machine Learning" (root task)
   → "Week 1: Linear Regression" (subtask)
      → "Watch Andrew Ng lecture" (sub-subtask)
      → "Implement in Python" (sub-subtask)
         → "Write gradient descent from scratch" (sub-sub-subtask)

Time Tracker

  • Every pending task appears here with a timer
  • Click Start to begin timing — the seconds tick live on screen
  • Click Stop to end the session — the seconds are saved to the database automatically
  • The total focus time for the session is shown at the top
  • The Pomodoro timer below lets you do focused 25-minute sessions
  • Click Focus 25m / Break 5m / Long Break 15m to switch modes
  • An alert fires when the timer reaches zero

Calendar

  • Shows a full month grid. Use ‹ and › to navigate between months
  • The calendar automatically highlights today's date
  • Blue dots on a date = tasks with that deadline
  • Click a date to see all tasks due that day on the right panel
  • Scroll down to see upcoming deadlines in order
  • Click + Add Event to add a custom event on any date with a colour (blue, green, amber, red, purple)
  • Events are saved to the database and shown as coloured dots on the calendar
  • Events for the current month are listed below the calendar grid

Roadmap

  • Create a high-level goal like "Become a Data Scientist" with a target date and duration in months
  • Click the goal to expand it, then click + Month to add a month (e.g. "Month 1: Python + Statistics")
  • Click the month to expand it, then click + Week to add a week (e.g. "Week 1: Python Basics")
  • Click the week to expand it, then click + Day to add a day topic (e.g. "Variables and loops")
  • Everything is collapsible and editable — click ✕ to delete any item
  • All data saves to PostgreSQL immediately

Projects

  • Create project containers for multi-task work like "EDA on Titanic Dataset"
  • When adding tasks, link them to a project via the Project dropdown
  • Each project card shows: total tasks, done tasks, time logged from timers, target date, and a progress bar

Notes

  • Save formulas, cheatsheets, code snippets, and concepts with a title, body, and tag
  • Pin important notes to the top by clicking the 📍 button
  • Search across all note titles and bodies in real time
  • Notes persist in PostgreSQL — never lost

Revision (Spaced Repetition)

  • Add any topic you want to review later (e.g. "Bayes Theorem", "Big-O notation")
  • Choose a tag and initial interval (Tomorrow, 3 days, 1 week, 2 weeks, 1 month)
  • When the due date arrives, the topic appears in the "Due for review" section highlighted in red
  • Click ✓ Done after reviewing — the system automatically schedules the next review further out using the Leitner box system (Box 1→2→3→4→5 = 1→3→7→14→30 days)
  • The box level is shown next to each topic so you can see your mastery level

Analytics

  • Daily hours bar chart — shows your logged study hours for the last 7 days
  • Time by subject doughnut — shows how your actual timer time is distributed across subjects (only fills in once you use the task timers)
  • Weekly completion line chart — shows your task completion rate percentage over recent weeks
  • Tag completion stacked bar — shows done vs pending tasks per subject tag

Insights

  • Auto-generated observations built from your real data
  • Shows: your most productive day of the week, which subject you spend most time on, current completion rate, total hours logged, overdue task alerts, and streak encouragement
  • Recent logs are shown at the bottom with mood emoji, hours, what you studied, and wins

8. Deploy Free on Render.com (Recommended)

Render gives you a free PostgreSQL database (90 days, then $7/mo) and a free backend (spins down after 15 min of inactivity). This is the easiest option to get your app online.

Step 1 — Push your code to GitHub

cd growthOS

# Initialize git
git init
git add .
git commit -m "Initial GrowthOS commit"

# Create a new repo on github.com (name it growthos), then:
git remote add origin https://github.com/YOUR_USERNAME/growthos.git
git push -u origin main

Step 2 — Create a PostgreSQL database on Render

  1. Go to render.com → Sign up with GitHub (free)
  2. Click New + → PostgreSQL
  3. Name: growthos-db, Region: pick closest to you (Singapore for India)
  4. Plan: Free
  5. Click Create Database
  6. Wait ~1 min, then copy the External Database URL from the Info tab — it looks like:
    postgresql://user:password@host.oregon-postgres.render.com/growthos_xxxx

Step 3 — Deploy the backend API

  1. Click New + → Web Service
  2. Connect your GitHub repo
  3. Settings:
    FieldValue
    Namegrowthos-api
    Root Directorybackend
    RuntimeNode
    Build Commandnpm install
    Start Commandnpm start
    PlanFree
  4. Scroll to Environment Variables and add:
    KeyValue
    DATABASE_URLPaste the External Database URL from Step 2
    JWT_SECRETAny long random string, e.g. grwthos_prod_secret_XYZ_123_abc
    NODE_ENVproduction
    FRONTEND_URLLeave blank for now — fill in after deploying frontend
  5. Click Create Web Service — deployment takes ~3 minutes
  6. Once live, test it: open https://growthos-api.onrender.com/health — you should see {"status":"ok"}
  7. Run migrations via the Render Shell tab: click Shell → type node db/migrate.js → press Enter

Step 4 — Deploy the frontend

  1. Click New + → Static Site
  2. Connect the same GitHub repo
  3. Settings:
    FieldValue
    Namegrowthos
    Root Directoryfrontend
    Build Commandnpm install && npm run build
    Publish Directorydist
  4. Add Environment Variable:
    KeyValue
    VITE_API_URLhttps://growthos-api.onrender.com
  5. Click Create Static Site
  6. Once deployed, copy the frontend URL (e.g. https://growthos.onrender.com)
  7. Go back to the backend service → Environment → update FRONTEND_URL to your frontend URL → Save → it will redeploy
Your app is now live and accessible from any device! Share the frontend URL with yourself on your phone too.

9. Deploy on Railway

Railway is slightly simpler than Render and has a $5/month free credit that usually covers a small app.

  1. Go to railway.app → Login with GitHub
  2. Click New Project → Deploy from GitHub repo
  3. Select your growthos repo
  4. Railway auto-detects Node.js. Set Root Directory to backend
  5. In the service settings, click Add Plugin → PostgreSQL — Railway provisions a database and injects DATABASE_URL automatically
  6. Set environment variables: JWT_SECRET, NODE_ENV=production, FRONTEND_URL
  7. Set Start Command to npm start
  8. Open the Railway shell and run: node db/migrate.js
  9. Deploy the frontend to Vercel: go to vercel.com → Import Git repo → set Root Directory to frontend → add VITE_API_URL env var → Deploy

10. Deploy on Ubuntu VPS (Full Control)

Use a DigitalOcean Droplet ($6/mo), Hetzner, or any Ubuntu 22.04 server.

On your server — run all of these commands:

# 1. Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 2. Install PostgreSQL
sudo apt install -y postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 3. Create the database and user
sudo -u postgres psql -c "CREATE DATABASE growthos;"
sudo -u postgres psql -c "CREATE USER growthos_user WITH PASSWORD 'yourpassword';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE growthos TO growthos_user;"

# 4. Install PM2 (keeps Node.js running forever) and Nginx
sudo npm install -g pm2
sudo apt install -y nginx

# 5. Clone your repository
git clone https://github.com/YOUR_USERNAME/growthos.git /var/www/growthos
cd /var/www/growthos

# 6. Setup backend
cd backend
cp .env.example .env
nano .env
# Fill in: DATABASE_URL, JWT_SECRET, NODE_ENV=production, FRONTEND_URL=https://yourdomain.com
npm install
node db/migrate.js

# 7. Start backend with PM2
pm2 start server.js --name growthos-api
pm2 save
pm2 startup  # Copy and run the command it gives you

# 8. Build frontend
cd ../frontend
npm install
VITE_API_URL=https://yourdomain.com npm run build
# Built files go into frontend/dist/

Nginx config — create this file:

sudo nano /etc/nginx/sites-available/growthos
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    # Serve React frontend
    root /var/www/growthos/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Proxy API calls to Express backend
    location /api {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }
}
# Enable the site
sudo ln -s /etc/nginx/sites-available/growthos /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx

# Add HTTPS (free with Let's Encrypt)
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
Your app is now live at https://yourdomain.com with auto-renewing SSL. PM2 ensures it restarts automatically if the server reboots.

12. Troubleshooting

❌ "Cannot connect to the server" / blank page

  • Make sure the backend is running: cd backend && npm run dev
  • Make sure the frontend is running: cd frontend && npm run dev
  • Check the terminal for error messages
  • Visit http://localhost:5000/health — if you see JSON, the backend is working

❌ "role postgres does not exist" or DB connection error

# On Ubuntu, the default user is postgres, not your system user:
sudo -u postgres psql
CREATE DATABASE growthos;
\q

# Then set your .env to use:
DATABASE_URL=postgresql://postgres@localhost:5432/growthos
# (no password needed on local Ubuntu install)

❌ Migration fails: "already exists"

This is safe — all tables use CREATE TABLE IF NOT EXISTS. You can re-run the migration anytime without harm. If you see a different error, drop and recreate the database:

psql -U postgres -c "DROP DATABASE growthos;"
psql -U postgres -c "CREATE DATABASE growthos;"
cd backend && npm run db:migrate

❌ "Module not found" on npm run dev

cd backend && npm install
cd ../frontend && npm install

❌ 401 Unauthorized errors after deploying

  • Make sure JWT_SECRET in production matches what was used to generate existing tokens
  • If you changed the secret, all users will need to log in again — this is normal and expected

❌ CORS errors in production (API calls blocked)

  • Set FRONTEND_URL in backend environment to your exact frontend URL
  • Example: FRONTEND_URL=https://growthos.onrender.com (no trailing slash)
  • After changing env vars on Render or Railway, the service will redeploy automatically

❌ Calendar not showing the current month

  • The calendar auto-initialises to new Date() — the current month on your device
  • Use the ‹ and › buttons to navigate — it fetches events from the database for that month each time

❌ Timer data disappearing on page refresh

The in-progress timer state (counting up live) is held in React state and resets on refresh — this is intentional. Always click Stop before refreshing — clicking Stop is what saves the seconds to the database. Partially-timed sessions that were not stopped are not saved, by design.
